# 安全政策

## 支持的版本

我们目前支持以下版本的安全更新：

| 版本 | 支持状态 |
| --- | --- |
| 1.0.x | :white_check_mark: |
| < 1.0 | :x: |

## 报告漏洞

我们非常重视安全问题。如果你发现了安全漏洞，请负责任地披露：

### 报告流程

1. **不要**在公共 Issue 中报告安全漏洞
2. 发送邮件到项目维护者（如果有的话）
3. 或者创建一个私有的 GitHub Security Advisory

### 报告内容

请在报告中包含以下信息：

- 漏洞的详细描述
- 重现步骤
- 潜在的影响
- 建议的修复方案（如果有）
- 你的联系信息

### 响应时间

- 我们会在 48 小时内确认收到你的报告
- 我们会在 7 天内提供初步评估
- 我们会在 30 天内提供修复计划或解释为什么不需要修复

### 安全更新

安全修复将通过以下方式发布：

1. 创建新的补丁版本
2. 在 GitHub Releases 中发布
3. 在 CHANGELOG.md 中记录（不包含敏感细节）
4. 通过 GitHub Security Advisories 通知

## 安全最佳实践

### 对于用户

1. **保持更新**: 始终使用最新版本
2. **权限控制**: 不要以管理员权限运行应用
3. **输入验证**: 注意执行的命令内容
4. **网络安全**: 在安全的网络环境中使用

### 对于开发者

1. **依赖管理**: 定期更新依赖包
2. **代码审查**: 所有代码变更都需要审查
3. **输入验证**: 验证所有用户输入
4. **权限最小化**: 使用最小必要权限

## 已知安全考虑

### 命令执行

- 应用可以执行系统命令，存在潜在风险
- 用户应该谨慎执行不熟悉的命令
- 应用不会自动执行任何命令

### 文件系统访问

- 应用可以访问用户有权限的文件和目录
- 遵循操作系统的权限控制
- 不会访问系统敏感文件

### 数据存储

- 用户设置存储在本地
- 不收集或传输用户数据
- 命令历史仅本地存储

## 安全功能

### 输入验证

- 命令参数验证
- 路径安全检查
- 特殊字符处理

### 权限控制

- 遵循平台权限模型
- 不请求不必要的权限
- 用户可控的访问范围

### 错误处理

- 安全的错误信息显示
- 不泄露敏感系统信息
- 优雅的异常处理

## 第三方依赖

我们使用以下主要依赖：

- `flutter`: Google 官方 UI 框架
- `process_run`: 进程执行库
- `shared_preferences`: 本地存储库
- `path`: 路径处理库

我们会：
- 定期检查依赖的安全更新
- 使用 `flutter pub audit` 检查已知漏洞
- 及时更新有安全问题的依赖

## 安全审计

### 自动化检查

- GitHub Dependabot 监控依赖漏洞
- CI/CD 流程中的安全扫描
- 代码质量和安全分析

### 手动审查

- 定期的代码安全审查
- 第三方安全评估（如果需要）
- 社区安全反馈
